NwHIN 1.0 Portfolio > Scenarios > Lab Results Delivery

A laboratory sends a patient’s lab results to the ordering physician:

A patient visits his primary care physician for a routine visit. His doctor determines that lab work must be performed and orders for the results to be sent to her electronically from the laboratory upon completion. Once the laboratory tests are complete, the lab constructs a report to send electronically to the ordering physician. The laboratory adheres to HL7 v 2.5.1 Laboratory Results Interface Implementation Guidance to structure the content of the patient’s Electronic Health Record (EHR).

LOINC and RELMA 5.5 Vocabulary and Code Set specifications are used as naming conventions for data within the EHR. Finally, to safeguard the patient’s laboratory results during transmittal via the Internet, the laboratory employs transport and security mechanisms: Certificate Discovery using DNS and LDAP, the Applicability Statement for Secure Health Transport, and x.509 for Certificates. Using its Direct email account, the laboratory sends the electronic lab results report to the Ordering Physician’s Direct email address.


NwHIN 1.0 specifications needed to implement this scenario:

1. Content Structure
Guidance that specifies how to structure health information to ensure proper exchange
  • HL7 v.2.5.1 Laboratory Results Interface Implementation Guide

2. Vocabulary & Code Sets
Specifications that identify common naming convention necessary for proper health information exchange
  • Logistical Observation Identifiers names and Codes (LOINC) 2.38
  • RELMA 5.5

3. Transport and Security
Mechanisms and processes that safely exchange health information over the Internet
  • Certificate Discovery using Domain Name System (DNS) and Lightweight Directory Access Protocol (LDAP)
  • Applicability Statement for Secure Health Transport
  • x.509 for Certificates
