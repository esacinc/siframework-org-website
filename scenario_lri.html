Delivering Lab Results

A laboratory sends a patient’s lab results to the ordering physician:

A patient visits a reference lab for blood work prior to an annual health checkup with his primary care physician. Once the tests are complete, the lab technician compiles the results and enters them into the reference lab’s EHR. The lab technician then electronically delivers the lab results report to the primary care physician.

 

Meaningful Use Stage 2 Objectives related to lab results: Incorporate clinical lab-test results into Certified EHR Technology as structured data. (Meaningful Use Menu Set 10 for EPs, Meaningful Use Menu Set 8 for EH/CAHs)


Common workflow steps for this scenario:
  A technician at the lab extracts the patient’s lab results report, which has been properly formatted according to the HL7 v.2.5.1 Laboratory Results Interface Implementation Guide, including using the Logistical Observation Identifiers names and Codes (LOINC) 2.38 vocabulary. The technician then authors a Direct message and attaches the lab results report. In order to select the primary care physician as the recipient, the technician accesses a search dialogue in the HISP's user interface, which queries a Provider Directory (adhering to the Certificate Discovery for Direct specification) for the right Direct address.
     
  The laboratory’s HISP, a contracted brokering agent responsible for the management of security and transport for directed exchange, retrieves the primary care physician’s digital certificate from the Provider Directory. The laboratory’s HISP uses this certificate to encrypt the message and sends it to the primary care physician’s HISP in accordance with the Applicability Statement for Secure Health Transport, which uses x.509 for certificates.
     
  The primary care physician’s HISP decrypts the message and routes it to the primary care physician.
     
  The primary care physician uses the HISP’s user interface to access the lab report in his Direct inbox.
